Pixelbooth Documentation

Pixelbooth is a local-first Raspberry Pi photobooth appliance for an 8-bit-themed Friday bar. The working concept is Pixel Portal: High Score Selfie Booth: a kiosk capture experience, event gallery, admin control plane, hardware adapters, render pipeline, and future sharing/printing/sync features running close to the booth.

This documentation site is for people who need to build, operate, or extend the project. It publishes the canonical Markdown content from .docs/, but the navigation is organized around practical project use rather than internal AI collaboration notes.

Current State​

The project currently has a deliberate monorepo skeleton:

  • apps/api: Python/FastAPI backend with fake and real camera adapter seams.
  • apps/web: React/TypeScript/Vite frontend for booth, gallery, and admin.
  • .docs: product, architecture, roadmap, and decision documentation.
  • docker-compose.yml: local development shape for API and web services.

Stage 2 and most Stage 2.5 foundations are in place. Stage 3 camera work is in progress, with capture session orchestration, media processing, lighting, marquee, deployment, sync, print, challenges, and observability staged after that.

Where To Start​

  • Local Development: run Pixelbooth locally, open the app surfaces, and run checks.
  • Raspberry Pi Setup: first boot, app setup, camera checks, and dev-stack run commands for the Pi.
  • Project Structure: understand how the backend, frontend, docs, and hardware seams fit together.
  • Operator Runbook: the practical booth/operator view, including what exists now and what is planned for field operation.
  • Roadmap: the staged implementation plan.

Main Product Surfaces​

  • Booth kiosk at /: guest-facing capture flow.
  • Gallery at /gallery: event media browsing.
  • Admin at /admin: diagnostics, settings, assets, overlays, Commander, remotes, and future setup/log views.
  • API at /api/v1: REST and WebSocket runtime contract.
