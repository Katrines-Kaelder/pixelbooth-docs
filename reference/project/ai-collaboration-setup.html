AI Collaboration Setup

Pixelbooth's AI collaboration layer keeps implementation work aligned with the architecture docs and staged roadmap. The root CLAUDE.md is the entry point for AI agents, and .claude/ contains the project rules, skills, and reviewer agents.

Responsibility Split​

  • Rules define enforceable constraints that apply during implementation.
  • Skills hold build-time domain knowledge for architecture, backend, frontend, hardware, and review workflows.
  • Agents review focused areas after implementation.
  • Codex-specific agents live under .codex/ when a workflow needs Codex branch, PR, or publishing conventions rather than generic Claude review behavior.
  • .docs/ remains the source of truth for architecture and product decisions.

Contributor Checklist​

  1. Read the documentation index and follow its read order.
  2. Identify the owning doc before changing behavior, APIs, hardware contracts, or product flows.
  3. Check the relevant project rule and skill in .claude/.
  4. Update docs before or alongside implementation.
  5. Add an ADR for significant hard-to-reverse decisions.

Docusaurus Ownership​

The Docusaurus documentation site is owned by Codex in the dedicated Docusaurus worktree unless the user explicitly reassigns it. Documentation-refresh branches use the docusaurus/YYYY-MM-DD-what-changed naming convention. Other coding agents may update existing .docs content for their tasks, but should not scaffold or modify Docusaurus configuration, dependencies, scripts, or deployment workflows.

For recurring documentation refreshes, use the docs-branch-updater Codex agent. It pulls main, discovers changes that need builder/operator documentation, updates .docs, verifies the Docusaurus build, and opens a PR into docs.
