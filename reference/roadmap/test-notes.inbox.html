Test Notes Inbox

Scratch space for findings from validation/test sessions. Type freely here while testing — formatting does not matter. Afterwards, run the known-issues-intake Codex agent (.agents/agents/known-issues-intake.md); it reads this file and normalizes each finding into a proper row in known-issues.md, then leaves the edits for you to review and commit.

Follow the test-plan.md sessions for what to test.

How To Write A Finding​

One block per finding, separated by --- or a blank line. Every field is optional — include what you have, skip what you don't:

  • Ref: an existing KI-###, or the test-plan session/stage (e.g. "Session 2" / "Stage 4").
  • Feature: the smallest feature involved (e.g. "video playback in gallery").
  • What happened: observed behavior, plus any artifact — log line, media id, diagnostic id, screenshot path, command output, measurement.
  • Result: pass / fail / partial / blocked.

The agent infers status, IDs, next action, and the owning doc from this. Don't worry about getting it "right" — rough is fine.

Findings​

First run​

No way to select which backend/camera you want to use. Cant switch from the (admittedly) primary 2 x Camera Module stack, to the usbcamera if it is present. USB-camera is detected, but not tested.

Live preview is not real time, it has a delay. Should be near instant, maybe using something else than mjpeg? Framerate also seems very low. The hardware can handle it, and the live preview is local, cant see why we are limiting ourselves.

The recording and processing animations + progress bars, just seems like some preset animation. They also tend to stop at 94%, then hang, until it moves to the next screen. This needs to be improved. For the gif/video modes, they need to actually reflect how much time is left.

Images are captured as 1280 x 720. Should be full resolution of the pi camera module as default (4608px × 2592px)

Even though exposure settings are deferred, the pi cameras needs to have autofocus settings and such turned on

Gallery doesnt react to swiping on the touchscreen when viewing an item from the gallery.

When clicking through item previews from the gallery, the header on the preview that informs the user about when the item was captured, and which kind changes before the media does (especially on videos and gifs). If media needs to load first, we need a loading animation, with a static preview of the item.

Shutting down the app doesnt actually seem to shut down the app. If I force quit by pressing ctrl+c again, the csi camera streams arent released, so I cant restart the app, without restarting the Pi.
