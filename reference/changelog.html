Changelog

All notable Pixelbooth changes are summarized here from the full git history.

This project does not currently use release tags, so entries are grouped by development stage and date range. The changelog was last refreshed against origin/main commit 9940a00 on 2026-06-17.

Unreleased​

Stage 4 media fixes and Stage 8 Phase 1/2 share delivery - 2026-06-16 to 2026-06-17​

Added​

  • Added the operator-facing media file-store layout, per-format capture-size contract, runtime-editable capture sizes, and ADRs covering the Stage 4 render backend, capture sizing, file-store layout, and editable capture dimensions.

Stage 8 Phase 1 — share/sync services

  • Added a durable Stage 8 sync outbox on the shared job queue, so render, sync, and future print work share the same retryable background-work primitive.
  • Added rclone-backed sync/share remotes for S3/MinIO, WebDAV, FTP, SFTP, and Google Drive, with secrets kept out of command arguments. Google Drive is supported as a backup/sync remote via rclone OAuth token settings; it is not the recommended public QR backend.
  • Added share-remote designation, link-preview settings, and the admin sharing panel for configuring the public QR/share path.

Stage 8 Phase 2 — QR slice

  • Added per-media share status, media.shared events, scannable qr.svg endpoints, and manual reshare support.
  • Added the static arcade sharepage in apps/sharepage plus a share-host setup helper that deploys the page and prints the admin values operators need.
  • Added live share QR panels in the kiosk Present screen and gallery detail, so the booth can move from "saving" to a scannable QR after the upload is ready.

Fixed​

  • Fixed several Stage 4 media quality issues: browser-playable H.264 video, Pillow-core overlay compositing, scanline intensity during active capture, camera release on shutdown, and operator-facing capture-size handling.
  • Fixed admin typography legibility through theme-scoped font tokens.

Changed​

  • Refreshed the roadmap, known-issues register, and architecture docs for the Stage 8 Phase 1 share/sync services and Phase 2 QR path, including ADR-0018 for cloud/self-hosted sharing.
  • Updated the Stage 4 render notes to align with the Pillow/ffmpeg core backend decision.

Stage 4 follow-ups and composed gallery polish - 2026-06-15​

Added​

  • Added a dedicated gallery entry point from the booth attract screen, making captured media easier for guests to browse without entering the capture flow.
  • Overhauled the composed-media gallery experience with richer cards, improved media details, metadata helpers, and presentation refinements for event-ready browsing.
  • Added a roadmap known-issues register so resolved and pending product risks can be tracked in one place.
  • Added a budget tracker spreadsheet and receipt archive for better project cost tracking.
  • Added project skills and memory notes for the next UI design pass and Stage 4 guest composed-UX planning.

Changed​

  • Refreshed Stage 4 documentation, roadmap status, and project status to reflect composed capture and gallery progress.
  • Recorded media and documentation follow-ups, including the resolution of the gallery redesign issue.
  • Pulled the latest main history into the media-docs follow-up branch.

Stage 4 composed capture and documentation publishing - 2026-06-14 to 2026-06-15​

Added​

  • Added ADR-0013 for composed-capture choreography and implemented per-format capture behavior in the session service.
  • Added composed-format guest capture UX across the booth and gallery.
  • Added generated OpenAPI types to the frontend and migrated handwritten API types to the generated contract.
  • Added event-scoped overlays and composed media formats on the backend.
  • Added ADR-0012 for composed media, event overlays, and OpenAPI type generation.
  • Linked hosted documentation from the repository README.

Changed​

  • Refreshed the Docusaurus site presentation with compact footer treatment, improved status views, fixed booth-preview scaling, and a clearer split between source docs and the published presentation site.
  • Updated roadmap and status docs for Stage 4 progress.
  • Kept the roadmap source more agent-friendly for future automation and review.
  • Updated documentation publishing workflows, including branch-triggered publishing, renamed workflow labeling, and deploy-key hardening.

Stage 4 media-processing foundation - 2026-06-14​

Added​

  • Added a media catalog, shared job queue, render pipeline, and visible media outputs as the first Stage 4 media-processing foundation.
  • Added overlay-aware rendering and the first guest gallery frontend.
  • Added the shared job-queue decision document.

Changed​

  • Advanced the project from camera/session foundations into media-processing infrastructure with separate original, preview, rendered, and gallery-visible concerns.

Docusaurus documentation site - 2026-06-14​

Added​

  • Added the Docusaurus documentation site under apps/docs for a curated public documentation view.
  • Added manual documentation publishing and later automated docs-branch publishing support.
  • Added a Codex docs branch updater agent.

Changed​

  • Hardened docs deployment key handling, including support for base64-encoded deploy key secrets.
  • Updated documentation action naming and roadmap status around the Docusaurus publishing flow.

Stage 3.6 booth visual fidelity - 2026-06-14​

Added​

  • Rebuilt the booth attract screen around the arcade visual handoff.
  • Rebuilt mode selection to match the handoff arcade design.
  • Rebuilt countdown, processing, and result/present screens for the booth flow.

Changed​

  • Disabled the guest review step to keep the capture path more direct.
  • Pulled current main changes into the booth visual-fidelity branch before it landed.

Stage 3.5 session orchestration - 2026-06-13 to 2026-06-14​

Added​

  • Added backend SessionService capture orchestration for the first capture session state machine.
  • Added the session-state kiosk flow so the guest UI can move through capture states instead of only previewing a camera feed.
  • Added kiosk end-to-end coverage for the capture session loop.
  • Added a Docusaurus planning and workflow split to separate docs-site work from product implementation work.

Changed​

  • Moved the booth from preview-only behavior toward an explicit session-driven capture workflow.

Stage 3 camera hardware foundations - 2026-06-12 to 2026-06-13​

Added​

  • Added a USB/V4L2 camera adapter with live MJPEG preview support.
  • Added a preview-only booth kiosk surface connected to the live MJPEG preview.
  • Added kiosk end-to-end coverage for the preview-only booth surface.
  • Added Raspberry Pi setup scripts and Pi access notes.
  • Added a test-capture affordance to verify that USB still capture works from the booth/admin flow.
  • Added a Picamera2 CSI dual-camera adapter for bare-metal Raspberry Pi OS.
  • Added Raspberry Pi Connect boot persistence in the first-boot script.
  • Added hardware budget tracking to the README.

Fixed​

  • Fixed USB preview blanking by making camera availability checks avoid taking over an already-open camera.
  • Stabilized USB capture by sharing one V4L2 capture instance instead of opening competing camera handles.
  • Made E2E ports overridable to reduce local dev-server collisions.

Changed​

  • Recorded Stage 3 progress, hardware validation notes, and known follow-ups in project memory.

Stage 2.5 persistence and admin control plane - 2026-06-12​

Added​

  • Added the SQLite persistence foundation for local metadata storage.
  • Added the admin gate, logging substrate, configuration service, and asset services.
  • Added overlay, command, and sync-remote services.
  • Added the admin control-plane frontend.

Changed​

  • Disabled automatic opening of the Serena web dashboard during development.

Planning, architecture, and design foundation - 2026-06-10 to 2026-06-12​

Added​

  • Added the initial Python and React-oriented .gitignore.
  • Added the initial project setup, architecture notes, definitions, and deliberate implementation scaffold.
  • Added roadmap sequencing fixes, ADRs 0003 through 0006, and the booth touchscreen design brief.
  • Added the booth touchscreen design prototype as a reference artifact.

Changed​

  • Corrected planning docs and connected the frontend reference more explicitly to the product direction.
  • Made the booth prototype the explicit design source for later roadmap stages.
