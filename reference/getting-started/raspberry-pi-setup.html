Raspberry Pi Setup

Pixelbooth currently has two Raspberry Pi helper scripts. They are practical bring-up tools for Stage 3 camera work, not the final production installer. Stage 7 owns the future production deployment, systemd/container packaging, Wi-Fi fallback mode, and opening-night smoke-test flow.

Script Overview​

ScriptRun asPurpose
pi-first-boot.shroot via sudoFirst boot OS preparation: system update, basics, hostname, SSH, optional firewall, unattended upgrades, and health report.
pi-pixelbooth-setup.shnormal userApp setup and camera-spike workflow: apt packages, backend venv, frontend deps, doctor checks, and dev run command.

Both scripts should be run from the repository root after reviewing their configuration and comments.

First Boot OS Preparation​

Use pi-first-boot.sh on a fresh Raspberry Pi OS install after confirming the editable settings at the top of the script:

sudo bash ./pi-first-boot.sh

The script can:

  • set the hostname, defaulting to pixelbooth;
  • enable SSH;
  • install basic tools such as git, curl, tmux, and build essentials;
  • optionally enable passwordless sudo for the current user;
  • optionally harden SSH after key login is confirmed;
  • optionally enable UFW;
  • enable unattended upgrades;
  • write a first-boot report to ~/firstboot-report.txt;
  • reboot when complete, depending on REBOOT_AFTER.

Review these variables before running:

NEW_HOSTNAME="pixelbooth"
ENABLE_UFW="false"
HARDEN_SSH="false"
INSTALL_BASICS="true"
ENABLE_UNATTENDED="true"
ENABLE_PASSWORDLESS_SUDO="true"
REBOOT_AFTER="true"

Do not set HARDEN_SSH=true until SSH key login has been tested, otherwise the Pi may become harder to access remotely.

App And Camera Setup​

Run pi-pixelbooth-setup.sh as the normal Pi user, not root:

./pi-pixelbooth-setup.sh setup
./pi-pixelbooth-setup.sh doctor
./pi-pixelbooth-setup.sh run

The setup command installs:

  • Pi camera and build dependencies through apt;
  • python3-picamera2 and rpicam-apps for CSI Camera Module 3 testing;
  • v4l-utils and OpenCV support for USB/V4L2 testing;
  • a backend virtual environment with --system-site-packages;
  • Node and pnpm through Corepack;
  • workspace dependencies with pnpm install --frozen-lockfile.

The script may add the user to the video group. If that happens, log out and back in, or reboot, before running camera checks.

Camera Backend Selection​

The default backend is Picamera2 for the normal/wide CSI Camera Module 3 setup:

./pi-pixelbooth-setup.sh run

Role-to-camera-index mapping can be overridden:

PIXELBOOTH_PICAMERA2_NORMAL_INDEX=0 \
PIXELBOOTH_PICAMERA2_WIDE_INDEX=1 \
./pi-pixelbooth-setup.sh run

For a single USB webcam:

PIXELBOOTH_CAMERA_BACKEND=usb \
PIXELBOOTH_USB_CAMERA_DEVICE=0 \
PIXELBOOTH_USB_CAMERA_ROLE=normal \
./pi-pixelbooth-setup.sh run

Doctor Checks​

Use doctor after setup and after hardware changes:

./pi-pixelbooth-setup.sh doctor

It reports:

  • Python, Node, pnpm, and backend venv availability;
  • picamera2 import status;
  • cv2 import status;
  • membership in the video group;
  • CSI cameras detected by Picamera2;
  • /dev/video* devices and v4l2-ctl output when available.

Running The Dev Stack On The Pi​

./pi-pixelbooth-setup.sh run

This starts:

  • API on 0.0.0.0:8000;
  • web dev server on 0.0.0.0:5173;
  • Vite proxying /api to the local API.

Open from another device on the same network:

  • Booth: http://<this-pi>:5173/
  • Admin: http://<this-pi>:5173/admin
  • Cameras API: http://<this-pi>:8000/api/v1/cameras

The admin secret defaults to the development value unless overridden through the environment.

Frontend Build​

The script can also build the web app:

./pi-pixelbooth-setup.sh build

This creates apps/web/dist, but the API does not serve that static build yet. Production packaging is still planned for Stage 7.

Important Limitations​

  • These scripts are for development, bring-up, and camera-spike validation.
  • They are not the final production installer.
  • The run command uses the Vite dev server and uvicorn directly.
  • Wi-Fi fallback, official deployment units, production Compose files, and opening-night smoke tests remain Stage 7 work.
