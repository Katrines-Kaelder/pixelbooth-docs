Overlay Designer Schema (.pixelbooth-overlay.json)

Overlays are first-class media assets and render instructions. V1 overlays are a single transparent PNG at output size (see admin-control-plane.md §Overlay Concept). This document defines the forward-looking designer definition so V1 PNG overlays do not become a dead end: a V1 PNG overlay is exactly a one-layer designer document, and the Stage 2.5 data model already records the fields the designer needs.

This is the schema only. The drag-and-drop designer UI is a later stage; nothing here requires it for V1.

Document shape​

{
  "schema": "pixelbooth-overlay/1",
  "id": "ovl_a1b2c3d4e5f6",
  "name": "Friday Bar Frame",
  "output": { "width": 1280, "height": 800 },
  "safe_area": { "x": 48, "y": 48, "width": 1184, "height": 704 },
  "layers": [
    {
      "id": "layer_frame",
      "type": "image",
      "asset_id": "ast_1122334455",
      "transform": { "x": 0, "y": 0, "width": 1280, "height": 800, "rotation": 0, "opacity": 1.0 },
      "locked": true,
      "hidden": false
    }
  ]
}

Fields​

  • schema — versioned schema id (pixelbooth-overlay/<n>); bumps on breaking changes.
  • id — stable overlay id (matches the overlays catalog row).
  • name — operator-facing label.
  • output — final render canvas size in pixels; overlays composite at this size.
  • safe_area — optional rectangle marking where faces/important content sit, so decorative layers avoid it. Advisory metadata, not enforced at render.
  • layers — ordered bottom→top. Each layer:
    • id — stable layer id.
    • type — image | text | logo | qr_placeholder | decoration.
    • asset_id — reference to an assets catalog record (for image/logo).
    • text — for text layers: { "value", "font_asset_id", "size", "color", "align" }.
    • transform — { x, y, width, height, rotation, opacity } in output pixels.
    • locked / hidden — editor state, preserved through round-trips.

V1 mapping​

A V1 PNG overlay is represented as:

  • output taken from the PNG dimensions (validated on upload).
  • a single image layer referencing the uploaded PNG asset_id, full-canvas transform, locked: true.

The Stage 2.5 overlays table stores source_type (png | designer), asset_id, and output_width/height; a future designer source stores the full JSON document above as the overlay definition. The render pipeline (Stage 4) reads either form through the same overlay record.
